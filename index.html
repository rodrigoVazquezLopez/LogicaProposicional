Instrucciones

La calculadora esta diseñada para realizar el cálculo de de proposiciones lógicas utilizando tablas de verdad. Las operaciones lógicas que se pueden realizar se muestran en la siguiente tabla:

Prioridad Operador Operación
1 ¬ Negación: ¬A
2 ∧ Conjunción: A∧B
3 ∨ Disyunción: A∨B
4 → Implicación: A→B
5 ↔ Bicondicional: A↔B
6 ⊕ Disyunción exclusiva: A⊕B

Se pueden utilizar paréntesis para agrupar las operaciones, sin embargo, en caso no hacerlo el orden de evaluación de los operadores se realiza con base en valor definido en la columna prioridad de la tabla anterior, siendo la negación la operación con mayor prioridad y la disyunción exclusiva la operación con menor prioridad.

Para ingresar las proposiciones utilice los botones con la letras A - I.

En caso que necesite corregir la expresión ingreada utilice el botón DEL para borrar los caracteres que considere necesarios.

Para borrar por completo el contenido pulse el botón AC.

Para visualizar el resultado pulse el botón RES.

Tokenizer

La expresión ingresada se analiza con la finalidad de identificar los distintos elementos que la componen: proposiciones, operaciones y paréntesis. Como resultado del análisis, la expresión se separa en tokens que se almacenan de manera independiente junto con una etiqueta que identifica el tipo.

Shunting Yard Algorithm

Una vez que se verificó que la cadena de entrada es sintácticamente correcta, el siguiente paso pepararla para evaluar la expresión. Por ello, la expresión originalmente ingresada en notación infija se debe convertir a notación prefija (también conocida como notación polaca inversa) con la finalidad de facilitar el cómputo.

Una de las formas de realizar la conversión infijo a posfijo es utilizar el algoritmo Shunting Yard el cuál es un algoritmo basado en el uso de una pila (stack).

A continuación se muestra el proceso de evaluaación del algoritmo con la cadena de entrada mostrando la cadena parcial de salida y el contenido de la pila de cada iteración.
